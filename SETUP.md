# MoC Cultural Digest: Daily Cloud Routine Setup

This repository powers the MoC Daily Cultural Digest, generated by
**Claude Code cloud routines** (claude.ai/code/routines). Each run is a
fully autonomous cloud session that clones this repo, sources news live from
the web for the last 24 hours, writes the digest, builds the Word document,
maintains continuity with prior editions, audits itself, and delivers.

Routines documentation: https://code.claude.com/docs/en/routines

## Prerequisites

- A claude.ai plan with Claude Code on the web (Pro, Max, Team, or Enterprise
  with eligible seats). Routines are in research preview; a Team/Enterprise
  admin must have the Routines toggle enabled.
- GitHub connected to your claude.ai account, with access to wherever you
  push this repo (run `/web-setup` from the CLI, or connect at claude.ai/code).
- (Optional) The **Dropbox connector** on your claude.ai account
  (claude.ai/customize/connectors), if Dropbox is the delivery channel you
  want — not required; the repo commit is the delivery floor regardless.
- (Optional but recommended) A **Tavily API key** for reading bot-walled
  premium wires (Reuters, FT, NYT, WaPo, Bloomberg, WSJ). Without it, those
  outlets still get verified via search snippets and second-source
  corroboration, just with thinner body-text detail.
- Merge this repo into `main` before scheduling: runs clone the default
  branch, so the skill and seeded `reports/` files must be there.

## 0. Push this repo to GitHub

This project isn't on GitHub yet. Create a new repository under an account
you control (or a dedicated service account — see the account-binding note
below) with `main` as the default branch, then push everything in this
folder to it.

## 1. Set up the Tavily reader (optional, API key only)

News sites behind Cloudflare refuse automated fetching from cloud IPs, so a
Tavily API key lets the routine retrieve article text reliably for the
handful of premium-wire articles that need it.

1. Create a Tavily account at app.tavily.com and copy an API key
   (`tvly-...`). The free tier (1,000 credits/month) comfortably covers
   daily runs — SKILL.md's credit-discipline rule keeps this to roughly 1-2
   credits per edition.
2. Add `TAVILY_API_KEY` to the routine's environment variables (see step 3
   below). Never commit the key to the repository.

Optional fallback: a reader MCP connector (Tavily, Exa, Firecrawl, or Jina
Reader) added at claude.ai/customize/connectors. SKILL.md auto-detects any
tool whose name contains `extract`, `scrape`, `reader`, or `read_url` and
uses it only when `TAVILY_API_KEY` isn't set.

## 2. Set up Dropbox delivery (optional)

Skip this entirely if the repo commit (plus, later, email) is enough for
now. If Dropbox delivery is wanted:

1. Go to dropbox.com/developers/apps -> **Create app** -> **Scoped access**
   -> pick the access type that reaches your destination folder -> name it
   something like `moc-digest-uploader`.
2. On the app's **Permissions** tab, tick `files.content.write` and
   `files.content.read`, then **Submit**.
3. Get a refresh token (one-time, in a browser + terminal):
   - Visit
     `https://www.dropbox.com/oauth2/authorize?client_id=<APP_KEY>&response_type=code&token_access_type=offline`
     while signed in to the right Dropbox account, and approve. Copy the
     code shown.
   - Exchange it:
     `curl https://api.dropbox.com/oauth2/token -d code=<CODE> -d grant_type=authorization_code -u <APP_KEY>:<APP_SECRET>`
     and copy the `refresh_token` from the response.
4. Add `DROPBOX_APP_KEY`, `DROPBOX_APP_SECRET`, `DROPBOX_REFRESH_TOKEN`, and
   `DROPBOX_DEST_FOLDER` (the destination path) to the routine's environment
   variables.

Refresh tokens don't expire on a schedule; this is set-and-forget unless the
app is revoked.

## 3. Create the environment

Routines inherit their environment's network policy, and the default
Trusted policy blocks arbitrary domains, which breaks fetching news
articles.

1. At claude.ai/code, open the environment selector (cloud icon) -> **Add
   environment**. Name it e.g. `cultural-digest`.
2. **Network access: Full.** The routine must fetch and verify articles from
   arbitrary news domains; a custom allowlist can't anticipate them.
3. **Environment variables**: `TAVILY_API_KEY` if using it, plus the three
   Dropbox variables and `DROPBOX_DEST_FOLDER` if using Dropbox delivery.

## 4. Create the daily routine

At claude.ai/code/routines, create a new routine pointed at this repo, on
whatever schedule fits (the original brief's default is a rolling 24-hour
window, so the specific time of day is a house preference, not a
correctness requirement). Suggested prompt:

> Run the cultural-digest skill for today's MoC Daily Cultural Digest,
> covering the last 24 hours. Follow SKILL.md exactly, including the
> Stage 5 audit gate — do not deliver a digest that fails the audit.

## 5. Test before trusting the schedule

Click **Run now** on the routine's detail page. Open the run session and
check: the digest committed to `main` (`reports/...md` + `.docx`), delivery
succeeded on whichever channel is configured, and the run log confirms every
audit check passed. Run it once before leaving it scheduled.

## Requesting revisions

Open the run's session at claude.ai/code (or start a fresh session on this
repo) and say what to change. The skill's revision handling follows the same
pattern as the conflict-monitoring project this was adapted from: a scoped
corrective round, written as the next draft (`_D2`, `_D3`, ...) without
overwriting earlier drafts, fully re-audited and redelivered, with the
instruction logged as an editorial learning.

## Reliability and limits

- **Green is not success.** The run log at the end of each session is the
  real verdict — it must confirm every audit check passed.
- **Usage limits.** Routine runs draw down the owning account's subscription
  usage and daily routine-run cap.
- **Account binding.** Routines act as the account that owns them — commits
  carry that account's GitHub identity, Dropbox uploads use its linked
  Dropbox. Use a dedicated service account if this shouldn't carry an
  individual's identity.
- **Org policies.** Organization IP allowlisting blocks all cloud sessions;
  ask Anthropic support for an exemption if your org uses it.

## Adjusting the routine

- **Change the schedule**: edit the routine's schedule trigger.
- **Change the spec** (sections, sourcing, tone, delivery): edit
  `.claude/skills/cultural-digest/SKILL.md` on `main`; the routine picks it
  up on its next run.
- **Add a real branded template**: see `templates/README.md`.
- **GitHub Actions alternative**: not built in this repo yet (unlike the
  sister project's parked `deploy/github-actions/`); add if a self-hosted
  cron fallback is ever wanted instead of routines.
